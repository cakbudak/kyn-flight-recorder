How Agent Studio works

The public product is intentionally smaller than Kyn.ist. Its contracts are not decorative.

What actually happens More than traces Runtime contract Deliberate boundary

Execution

What actually happens

A Flow pins exact Action or Agent versions. The runtime validates its input, maps it into each Step, invokes every capability through one Action boundary, commits receipts and hash-linked events, and follows only declared routes. External model I/O happens outside SQLite write transactions.

Input→Validate→Execute→Receipt→Route

Control plane

Why this is more than a trace viewer

You define executable Actions and compose Flows here. Runs can genuinely pause for Human approval, resume into a bounded effect, fail terminally, and rerun as linked immutable children. The Repair Lab adds evidence-owned diagnosis and a revision-fenced repair loop.

Contracts

Definitions and Runs stay distinct

Definition plane

Actions, Flows, Agents, Prompts, and Skills are immutable, fingerprinted versions.

Execution plane

Runs, Steps, model calls, receipts, approvals, effects, and events are authoritative records.

Authority plane

Skills grant exact Action version IDs. Model text never widens authority.

Maintenance plane

Terminal Runs remain absorbing. A rerun is a linked new Run against a pinned Flow version.

Safety

What is deliberately bounded

This public projection excludes Ainou, Parts/Entities, Bricks/Packs/Frames, CE, arbitrary MCP servers, shell access, filesystem access, arbitrary HTTP, and production connectors. The only write effect appends an idempotent record to this workspace’s SQLite sandbox. That keeps the demonstration real without publishing Kyn.ist’s private architecture.
